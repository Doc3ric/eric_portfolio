## Overview
The School Management System is a centralized financial tracking platform designed to streamline fee collections, track payment statuses, and automate reporting across multiple school years. Built to replace inefficient manual processes, the application provides administrators with real-time financial visibility and robust reporting tools, significantly reducing administrative overhead.

## The Problem
Prior to this application, the school relied on fragmented spreadsheets and paper-based ledgers to manage PTA fee collections. This manual approach made it incredibly difficult to maintain accurate payment histories, identify parents with outstanding balances, and transition financial records smoothly between academic years. Generating end-of-term financial reports was a tedious, error-prone process that consumed hours of administrative time.

## The Solution
I developed a comprehensive, automated fee management system that digitizes the entire collection lifecycle. The application allows administrators to easily configure academic years, securely log payments, and instantly generate structured PDF and Excel reports for collected payments and unpaid dues. By centralizing the data into a single source of truth, the system eliminated data discrepancies and provided a clear, auditable trail for all financial transactions.

## Technical Challenges & Architecture
- **Performance-Optimized Reporting:** Engineered efficient data export pipelines to handle large administrative datasets. Generating real-time lists of unpaid parents required writing optimized, complex SQL queries to cross-reference total enrollments against payment ledgers without blocking the main application thread.
- **Temporal Data & Schema Design:** Architected a robust relational database schema to seamlessly handle dynamic school year transitions. Financial records were carefully isolated by academic year, ensuring that historical data remained immutable for auditing purposes while current records remained highly accessible.
- **Separation of Concerns for Data Pipelines:** Built custom export controllers (`PaymentsExport`, `UnpaidParentsExport`) to abstract complex data formatting logic away from the UI. This maintained strict MVC principles and ensured the system could quickly generate reliable, multi-format (PDF/Excel) financial summaries.

## Lessons Learned
This project reinforced the importance of writing highly optimized database queries when dealing with financial reporting, and demonstrated how automating previously manual data entry can deliver immediate, measurable business value to an organization.
